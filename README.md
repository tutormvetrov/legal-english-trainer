# Legal English Trainer

Оффлайн desktop-приложение для тренировки юридического английского с интервальным повторением, несколькими учебными режимами, статистикой, импортом своих терминов и локальной активацией.

Проект написан на `PyQt6`, использует `SQLite` для данных и работает без Docker и без серверной части.

## Что умеет приложение

- интервальное повторение по `SM-2`
- карточки, сопоставление, диктант, детектив, контекст
- избранное и поиск по терминам
- аркадный `Boss Mode`
- мини-квиз после серии карточек
- статистика, weak terms, экспорт CSV
- импорт своих терминов из `CSV`, `JSON` или вставкой текста
- резервное копирование и восстановление базы
- светлая и тёмная темы
- локальная активация без сервера

## Структура проекта

- `run.py` — безопасная точка входа с crash logging
- `src/main.py` — инициализация приложения
- `src/gui/` — интерфейс и режимы
- `src/database/db_manager.py` — работа с SQLite
- `src/algorithms/spaced_repetition.py` — логика SM-2
- `src/utils/license_manager.py` — локальная активация
- `tests/` — unit/smoke tests

## Как запустить проект из исходников

1. Установить зависимости:

```powershell
pip install -r requirements.txt
```

2. Запустить приложение:

```powershell
python run.py
```

3. При первом запуске ввести ключ активации и имя пользователя.

Для локальной разработки ключ можно сгенерировать так:

```powershell
python scripts/keygen.py
```

Скрипт печатает валидные ключи в консоль. При желании он может сохранить их в `~/.letapp/generated_keys.txt`, а не в репозиторий.

## Как теперь вообще запускать приложение

Есть два нормальных сценария.

### Вариант 1: запуск из исходников

```powershell
pip install -r requirements.txt
python run.py
```

Это основной путь для разработки.

### Вариант 2: запуск собранного `.exe`

Сначала собрать:

```powershell
pyinstaller --noconfirm LegalEnglishTrainer.spec
```

Потом запускать файл:

`dist/LegalEnglishTrainer.exe`

## Где хранятся данные пользователя

Приложение использует директорию `~/.letapp`.

Там лежат:

- локальная активация
- streak
- highscore
- crash log
- пользовательская БД в packaged-режиме

При запуске из исходников база по умолчанию хранится в:

`data/legal_english.db`

## Активация

Активация полностью локальная, но теперь сохранённая сессия:

- подписывается
- привязывается к текущей машине
- хранится не в открытом `keys.txt`

Это не серверная защита и не DRM, но простой перенос activation-файла между машинами больше не является штатным сценарием.

Если нужно сбросить локальное состояние приложения:

```powershell
python scripts/reset_app.py
```

## Учебные режимы

### Карточки

Главный режим с SM-2.

- `Показать перевод`
- `Знаю`
- `Не знаю`
- `Пропустить`
- мини-квиз каждые `N` карточек

### Сопоставление

Сопоставление терминов и переводов. Верные пары пишутся в прогресс.

### Диктант

Ввод ответа с 3 попытками. Качество ответа влияет на SM-2.

### Детектив

Угадывание термина по определению с подсказками и очками. Результат пишет в SM-2.

### Контекст

Восстановление пропущенного термина по примеру предложения.

Важно:

- режим теперь интегрирован в `SM-2`
- сначала выбирает due-термины с примерами
- затем уже fallback на обычные термины с примерами

### Избранное

Подборка starred-терминов с поиском и карточкой деталей.

### Boss Mode

Аркадный режим на скорость.

Важно:

- `Boss Mode` не влияет на `SM-2`
- `Boss Mode` не влияет на дневной прогресс
- это осознанное отдельное правило

### Статистика

- summary cards
- прогресс по категориям
- weak terms
- экспорт прогресса
- импорт терминов
- backup / restore
- reset

## Импорт данных

Поддерживаются:

- `CSV`
- `JSON`
- вставка текста с настраиваемыми разделителями

Обязательные поля:

- `term_eng`
- `term_rus`

Дубликаты по `term_eng` игнорируются. В интерфейсе показывается реальное число добавленных и пропущенных строк.

## Backup / Restore

Резервная копия делается через `SQLite backup API`, а не через простое копирование открытого файла БД.

Восстановление:

- валидирует файл
- подменяет БД через временный файл
- не оставляет приложение в полусломанном состоянии при ошибке

## Поиск

Поиск терминов в БД теперь работает корректно как для латиницы, так и для кириллицы, без зависимости от особенностей `SQLite LIKE`.
